# Fas 0.5 — Söndag-matchning i boka.html

**Status:** KOD-VERIFIERAD (empirisk test ej möjlig med nuvarande testdata)
**Commit:** fa8e4c0

---

## Vad fix:en gör

Konverterar JS `getDay()` (0-6, sön=0) till ISO `day_of_week` (1-7, sön=7) innan jämförelse mot `cleaner_availability_v2.day_of_week`.

---

## Kod-bevisning ([boka.html:1359](../../boka.html:1359))

```js
const jsDay = new Date(state.date + 'T00:00:00').getDay();
const dow = jsDay === 0 ? 7 : jsDay;
```

Samma mönster på [boka.html:1893](../../boka.html:1893) — samma konvertering före `a.day_of_week === dow`-jämförelse i filter-blocket.

---

## Varför empirisk test ej genomfördes ikväll

Söndag 2026-04-26 från Stockholm Odenplan förväntas ge:

- RPC `find_nearby_cleaners` returnerar 5 cleaners
- Söndag-availability-filter (`dow=7`) reducerar till 2 (Farhad + Rafael)
- Payment-readiness-filter (`status='aktiv'`) reducerar till 1 (Farhad)

Endast 1 förväntad cleaner = svagt bevis (tom lista skulle inte kunna skiljas från söndag-bugg vs filter-bugg).

---

## Indirekt bevisning

Samma konverterings-mönster (`jsDay === 0 ? 7 : jsDay`) användes i `cleaner-job-match` EF (commit b7c9c8b) med full empirisk verifiering:

- Mån/tis: 12/12 cleaners med `avail > 0`
- Lör/sön: 7/12 cleaners med `avail > 0` (5 saknar helg-rader i v2)

Datakällan är samma (`cleaner_availability_v2`), filter-logiken är samma. Fas 0.5 vilar på samma fundament.

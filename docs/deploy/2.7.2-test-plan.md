# §2.7.2 Manual test-plan

**Fil:** [`boka.html`](../../boka.html)
**Scope:** UI + state + validering + submit-payload för B2B-insamling. Ingen EF-mottagning (det är §2.7.3 — `booking-create` uppdateras senare för att faktiskt spara fälten).

**Viktigt:** fram till §2.7.3 landat kommer booking-create-EF **ignorera** de 6 nya payload-fälten (de kastas när Supabase ser okänd kolumn i INSERT, beroende på behavior). Kör därför testerna i **preview/staging**, inte mot prod. Alternativt: verifiera bara att `console.log(payload)` innehåller korrekt data — stoppa innan Stripe Checkout triggas.

## Test 1 — Privat-flow regression (ingen B2B)

**Syfte:** säkerställ att ingen regression införts i befintliga B2C-flödet.

1. Öppna `boka.html` (default: customer_type='privat')
2. Verifiera:
   - [ ] `#business-fields` har `display:none`
   - [ ] `#pnr-wrap` är synlig
   - [ ] RUT-toggle synlig och är på
3. Fyll i privat-bokning till slutet (utan submit)
4. Klicka submit → fyll i PNR
5. Verifiera i console:
   ```js
   // Payload ska innehålla:
   customer_type: 'privat',
   business_name: null,
   business_vat_number: null,
   business_contact_person: null,
   invoice_address_street: null,
   // ...alla business_* + invoice_* är null
   rut: true  // om useRut är aktivt
   ```

**Godkänt om:** payload ser exakt ut som tidigare + alla 6 nya fält är `null`.

## Test 2 — Foretag-flow med minimal data

**Syfte:** verifiera att B2B-UI visas korrekt och obligatoriska fält räcker.

1. Klicka "🏢 Företag"-knappen (eller öppna `?type=foretag`)
2. Verifiera:
   - [ ] `#business-fields` synlig
   - [ ] `#pnr-wrap` dold
   - [ ] RUT-toggle dold
   - [ ] H1 ändrad till "Boka företagsstädning online"
   - [ ] Sidebar: "Exkl. moms" istället för RUT-rad
3. Fyll i bara obligatoriska:
   - Företagsnamn: `Acme AB`
   - Org.nr: `556677-8899`
4. Lämna alla nya fält tomma (momsreg, kontakt, faktura-email, fakturaadress-checkbox omarkerad, referens)
5. Fortsätt till slutet → klicka submit
6. Verifiera i console att payload innehåller:
   ```js
   customer_type: 'foretag',
   business_name: 'Acme AB',
   business_org_number: '556677-8899',
   business_vat_number: null,
   business_contact_person: null,
   business_invoice_email: null,
   business_reference: null,
   invoice_address_street: null,
   invoice_address_city: null,
   invoice_address_postal_code: null,
   rut: false
   ```

**Godkänt om:** obligatoriska fält ifyllda + alla valfria är `null`.

## Test 7 — Validering fakturaadress (required om checkbox)

**Syfte:** verifiera conditional required.

1. Toggle checkbox OFF → alla 3 adressfält tomma → submit: **passerar**
2. Toggle checkbox ON → alla 3 tomma → submit: **3 fel** (street, postal, city)
3. Toggle ON, street=`Gat` (för kort) → **fel** (minst 5 tecken + siffra)
4. Toggle ON, postal=`17164` (utan mellanslag) → **passerar** (regex tillåter `^\d{3}\s?\d{2}$`)
5. Toggle ON, postal=`1716` → **fel**

## Test 8 — PNR-hiding (Regel #26 C)

**Syfte:** säkerställ GDPR-compliance.

1. Starta som foretag
2. Verifiera `#pnr-wrap` är dold
3. Öppna dev-tools → Elements → sök `pnr-wrap` → verifiera `style="display:none"`

## Kända begränsningar

1. **Booking-create-EF ignorerar nya kolumner fram till §2.7.3.** Om du kör full submit mot prod innan dess, bokningen skapas utan de nya B2B-fälten. Säkrast att stoppa vid `console.log(payload)`.
2. **Ingen serverside validering** ännu — rent klient-side i §2.7.2. En användare med inaktiverat JS eller som modifierar DOM kan skicka ogiltig data. Hanteras i §2.7.3 (booking-create).
3. **DB-kolumnerna finns** (§2.7.1 körd i prod 2026-04-23) men EF:n mappar dem inte till payload ännu.

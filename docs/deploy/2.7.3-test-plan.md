## Test 1 — Privat-bokning regression (ingen B2B)

**Syfte:** säkerställ att B2C-flödet fortsätter fungera utan regression.

1. Öppna `boka.html` i browser
2. Default: customer_type='privat'
3. Fyll i privat-bokning med alla fält + RUT + PNR
4. Submit → Stripe Checkout → betala med test-kort `4242 4242 4242 4242`
5. Studio SQL — verifiera senaste bokningen:

```sql
SELECT
  id, customer_type,
  business_name, business_org_number, business_reference,
  business_vat_number, business_contact_person, business_invoice_email,
  invoice_address_street, invoice_address_city, invoice_address_postal_code,
  rut_amount, rut_application_status
FROM bookings
WHERE customer_email = 'farrehagge@gmail.com'
ORDER BY created_at DESC
LIMIT 1;
```

**Godkänt om:**
- `customer_type = 'privat'`
- Alla 9 business_*/invoice_*-kolumner = `NULL`
- `rut_amount > 0` (om RUT aktivt)
- `rut_application_status = 'pending'`

## Test 4 — customer_type-validering (hybrid)

**Syfte:** verifiera B-10-beslut (default privat om saknas, 400 om ogiltigt satt).

**Test 4a: customer_type saknas** (backwards-compat):

```powershell
$body = @{
  name = "Testare"
  email = "test4a@spick.se"
  phone = "0701234567"
  address = "Testvägen 1"
  date = "2026-05-15"
  time = "10:00"
  hours = 3
  service = "Hemstädning"
  sqm = 65
  # customer_type UTELÄMNAS helt
} | ConvertTo-Json

Invoke-RestMethod -Uri "https://urjeijcncsyuletprydy.supabase.co/functions/v1/booking-create" `
  -Method POST -Body $body -ContentType "application/json" `
  -Headers @{ apikey = $env:SUPABASE_ANON_KEY; Authorization = "Bearer $env:SUPABASE_ANON_KEY" }
```

**Förväntat:** 200 OK + bokning skapad med `customer_type='privat'`.

**Test 4b: customer_type = 'invalid'** (bad-actor-skydd):

Samma payload som 4a, men `customer_type = "invalid"`.

**Förväntat:** 400 Bad Request med felmeddelande:
```json
{ "error": "Ogiltigt customer_type: 'invalid'. Tillåtna värden: 'privat' eller 'foretag'." }
```

**Test 4c: customer_type = null** (backwards-compat explicit null):

`customer_type = $null`.

**Förväntat:** 200 OK + `customer_type='privat'`.

**Test 4d: customer_type = ''** (backwards-compat explicit tom sträng):

`customer_type = ""`.

**Förväntat:** 200 OK + `customer_type='privat'`.

## Test 5 — Sanitering + null-tvång (B-11)

**Syfte:** verifiera att `customer_type='privat'` tvingar alla business_* → NULL även om frontend skickar värden (data-integritet mot cache-drift).

**Scenario:** gammal/manipulerad client skickar:
```json
{
  "customer_type": "privat",
  "business_name": "Acme Ghost AB",
  "business_vat_number": "SE559402452201"
}
```

**Förväntat:** 200 OK, bokning skapad, men i DB:
- `customer_type = 'privat'`
- `business_name = NULL`  ← **tvingad null**
- `business_vat_number = NULL`  ← **tvingad null**

PowerShell:
```powershell
$body = @{
  name = "Test5"; email = "test5@spick.se"; phone = "0701234568"
  address = "Testvägen 5"; date = "2026-05-15"; time = "11:00"
  hours = 3; service = "Hemstädning"; sqm = 65
  customer_type = "privat"
  business_name = "Acme Ghost AB"
  business_vat_number = "SE559402452201"
} | ConvertTo-Json

Invoke-RestMethod -Uri "https://urjeijcncsyuletprydy.supabase.co/functions/v1/booking-create" `
  -Method POST -Body $body -ContentType "application/json" `
  -Headers @{ apikey = $env:SUPABASE_ANON_KEY; Authorization = "Bearer $env:SUPABASE_ANON_KEY" }
```

Verifiera i Studio: `SELECT customer_type, business_name, business_vat_number FROM bookings WHERE customer_email='test5@spick.se'`.

## Test 6 — Whitespace-sanitering (B-12)

**Syfte:** verifiera att whitespace-only strängar → NULL.

**Scenario:** `customer_type='foretag'`, `business_name='   '` (3 mellanslag).

**Förväntat:** `business_name = NULL` i DB (sanitize-helper trim:ar + returnerar null).

PowerShell:
```powershell
$body = @{
  name = "Test6"; email = "test6@spick.se"; phone = "0701234569"
  address = "Testvägen 6"; date = "2026-05-15"; time = "12:00"
  hours = 3; service = "Kontorsstädning"; sqm = 100
  customer_type = "foretag"
  business_name = "   "  # whitespace-only
  business_org_number = "556677-8899"
} | ConvertTo-Json

Invoke-RestMethod -Uri "https://urjeijcncsyuletprydy.supabase.co/functions/v1/booking-create" `
  -Method POST -Body $body -ContentType "application/json" `
  -Headers @{ apikey = $env:SUPABASE_ANON_KEY; Authorization = "Bearer $env:SUPABASE_ANON_KEY" }
```

Verifiera i Studio: `business_name` ska vara `NULL`, inte `'   '`.

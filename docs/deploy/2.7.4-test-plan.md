## Test 7 — Idempotens B2B (E3 — INGEN regenerering av F-nummer)

**Syfte:** verifiera att re-run av generate-receipt INTE förbrukar nytt sekvens-värde.

1. Gör en B2B-bokning (Test 3 eller 4), vänta tills mejl kommit.
2. Notera `invoice_number` från Studio (t.ex. `F-2026-00003`).
3. Trigga generate-receipt manuellt:

```powershell
$body = @{ booking_id = "<boking-id>" } | ConvertTo-Json
Invoke-RestMethod `
  -Uri "https://urjeijcncsyuletprydy.supabase.co/functions/v1/generate-receipt" `
  -Method POST -Body $body -ContentType "application/json" `
  -Headers @{ apikey = $env:SUPABASE_ANON_KEY; Authorization = "Bearer $env:SUPABASE_ANON_KEY" }
```

4. Förväntad response:
```json
{
  "success": true,
  "already_sent": true,
  "is_b2b": true,
  "document_number": "F-2026-00003",
  "receipt_url": "..."
}
```

5. Studio SQL: `invoice_number` oförändrat (fortfarande `F-2026-00003`).
6. Nästa verkliga B2B-bokning: sekvensen **ska** vara `F-2026-00004` (inte `00005`).

**Godkänt om:** ingen nytt nummer genererat, ingen sekvens-läckage.

## Test 8 — Serve-invoice F-regex (B-14)

**Syfte:** verifiera 1-rads regex-fix i serve-invoice.

1. Hitta en fakturanamn från Test 3 eller 4: `F-2026-NNNNN.html`
2. Öppna direkt:
   ```
   https://urjeijcncsyuletprydy.supabase.co/functions/v1/serve-invoice?file=F-2026-NNNNN.html
   ```
3. **Förväntat:** HTML-sida renderas med FAKTURA-layout.

**Pre-fix beteende (före §2.7.4):** `Invalid file` HTTP 400 (regex matchade bara `SF|KV`).

**Också testa regression:**
- `?file=KV-2026-00001.html` → KVITTO renderas ✓
- `?file=SF-2026-0001.html` → Städar-självfaktura renderas ✓ (om finns)
- `?file=BADPREFIX-2026-00001.html` → 400 Bad Request ✓

# §2.7.3 Deploy-instruktion

**Fil:** [`supabase/functions/booking-create/index.ts`](../../supabase/functions/booking-create/index.ts)
**Commit:** se `git log --grep="§2.7.3" --oneline`
**Skiljer sig från §2.5-R2:** booking-create är inkluderad i `deploy-edge-functions.yml` → auto-deploy via GitHub Actions. Ingen manuell `supabase functions deploy`-körning behövs.

---

## Auto-deploy-flöde

1. Commit + push till `main`
2. GitHub Actions triggar `.github/workflows/deploy-edge-functions.yml` (path-filter `supabase/functions/**`)
3. Workflow deployar alla 32 EFs listade i `FUNCTIONS`-array, inklusive `booking-create`
4. Verifiera i Actions-tab:
   - https://github.com/Spickapp/spick/actions
   - Hitta senaste "Deploy Edge Functions"-körning
   - Kolla att `--- Deploying booking-create ---` följs av `OK: booking-create`

## Verifiera deploy-status i CLI

```powershell
npx supabase functions list --project-ref urjeijcncsyuletprydy |
  Select-String "booking-create"
```

**Förväntat:** en rad med `booking-create` + senaste deploy-tidpunkt (ska vara efter commit-tidpunkt).

## Rollback-plan

Om deploy krashar produktion (payload-parsing-fel, TypeError, etc.):

### Alt A: Revert commit
```powershell
git revert <commit-hash>
git push origin main
# → GitHub Actions re-deployar föregående version automatiskt
```

### Alt B: Manuell rollback via Supabase CLI
```powershell
# Checkout föregående version av booking-create
git checkout <föregående-commit> -- supabase/functions/booking-create/index.ts
# Manuell deploy (om Actions inte triggas)
npx supabase functions deploy booking-create --project-ref urjeijcncsyuletprydy
```

**Trygghet:** §2.7.3 är strikt additiv (nya kolumner, ny validering). Rå regression-risk låg eftersom:
- §2.7.2 redan landat (frontend skickar 6 nya fält sedan 2026-04-23)
- DB-kolumner finns (§2.7.1)
- Befintlig B2C-path bevaras (customer_type='privat' → alla business_* NULL)

## Post-deploy Studio SQL verifikation

Efter att första nya B2B-bokning skapats (se [2.7.3-test-plan.md](2.7.3-test-plan.md) Test 3):

```sql
-- Verifiera att alla 10 B2B-kolumner populerats korrekt
SELECT
  id,
  customer_type,
  business_name,
  business_org_number,
  business_reference,
  business_vat_number,
  business_contact_person,
  business_invoice_email,
  invoice_address_street,
  invoice_address_city,
  invoice_address_postal_code,
  created_at
FROM bookings
WHERE customer_type = 'foretag'
ORDER BY created_at DESC
LIMIT 5;
```

Om kolumnerna populeras korrekt → §2.7.3 är klar.
